## 1. 設計方針

### 1-1. デザイン原則
- **シンプル第一**: 非エンジニアの社員160名が迷わず使える
- **モバイルファースト**: スマホでも快適に操作可能
- **視認性重視**: ステータスや進捗が一目でわかる
- **最小クリック**: 目的の画面に最短でたどり着ける

### 1-2. カラーパレット（コラボハウスブランド準拠）

**ブランドカラー（ロゴSVGから抽出）:**
- ブランドオレンジ: `#F99900` — コラボハウスのシンボルカラー
- ブランドチャコール: `#302D2C` — ロゴテキストの色
- ブランドオリーブ: `#574E2E` — ロゴ補助要素

**アプリ用カラーパレット:**
| 用途 | カラー | HEX | 説明 |
|------|--------|-----|------|
| プライマリ | コラボオレンジ | `#F99900` | メインアクション・ボタン・リンク |
| プライマリ（濃） | ダークオレンジ | `#E08800` | ホバー・アクティブ状態 |
| プライマリ（淡） | ライトオレンジ | `#FFF7EB` | 選択行・ハイライト背景 |
| テキスト（主） | チャコール | `#302D2C` | 見出し・本文テキスト |
| テキスト（副） | オリーブブラウン | `#574E2E` | サブテキスト・ラベル |
| テキスト（薄） | グレーブラウン | `#8C7E6A` | プレースホルダー・補助テキスト |
| 背景 | ウォームホワイト | `#FAFAF8` | ページ背景 |
| 背景（カード） | ホワイト | `#FFFFFF` | カード・パネル背景 |
| ボーダー | ライトベージュ | `#E8E4DE` | 区切り線・枠線 |
| サイドバー | チャコール | `#302D2C` | サイドバー背景（ロゴ色） |
| サイドバーテキスト | オフホワイト | `#F5F0EB` | サイドバー内テキスト |
| 成功 | グリーン | `#16A34A` | 完了・達成（green-600） |
| 警告 | アンバー | `#F59E0B` | 注意・リマインド（amber-500） |
| エラー | レッド | `#DC2626` | 未完了・問題（red-600） |
| 情報 | ブルー | `#2563EB` | お知らせ・ヒント（blue-600） |

**配色の考え方:**
- コラボハウスのオレンジ `#F99900` をプライマリカラーに採用し、ブランドの暖かさ・親しみやすさを表現
- テキストはロゴと同じチャコール `#302D2C` で統一感を持たせる
- サイドバーをチャコール背景にしてロゴが映えるデザイン
- 背景はウォームホワイト系にして全体を暖色トーンで統一
- ステータス色（成功/警告/エラー/情報）は視認性重視で標準色を使用

## 2. ロール別ユーザーフロー

### 2-1. 被評価者（employee）の主なフロー

```
ログイン → ダッシュボード（自分のタスクが見える）
  ├── 「目標を設定する」→ 目標設定シート入力 → 提出
  ├── 「行動評価を入力する」→ 依頼された他者の行動評価入力
  ├── 「評価結果を見る」→ 公開済みの自分の評価シート閲覧
  └── 「過去の賞与を見る」→ 賞与履歴一覧
```

### 2-2. 評価者（evaluator）の主なフロー

```
ログイン → ダッシュボード（配下メンバーの状況が見える）
  ├── 「評価シートを入力」→ メンバー選択 → 評価シート編集
  │     └── 各タブ: 成果 / Leader Radar / 能力 / 行動 / チェックリスト / 総評
  ├── 「360度評価を設定」→ メンバーごとに評価者3名を選出 → 通知送信
  ├── 「目標を確認・確定」→ メンバーの目標設定を確認 → 確定（ロック）
  ├── 「面談記録を書く」→ 面談記録入力
  └── 「評価結果を公開」→ 確定済み評価の公開操作
```

### 2-3. 管理者（admin）の主なフロー

```
ログイン → ダッシュボード（全社の進捗サマリー）
  ├── 「評価期間を管理」→ 新規期間作成 / 締切設定
  ├── 「全社の評価状況」→ 一覧で進捗確認（未入力者ハイライト）
  ├── 「レベル変更候補」→ 自動リストアップ → 承認/却下
  ├── 「賞与データ取込」→ Excelアップロード → プレビュー → 確定
  ├── 「社員データ管理」→ 一覧 / CSVインポート
  └── 「データエクスポート」→ 各種データをCSV出力
```

## 4. インタラクション設計

### 4-1. 自動保存
- 評価シートの各入力は**変更から2秒後に自動保存**（デバウンス）
- 保存完了時に「✓ 保存しました」をフッターに小さく表示
- オフライン時は「⚠ 保存できません」を表示

### 4-2. ロック・アンロック
- ロックされたセクションはグレーアウト＋🔒アイコン
- 入力フィールドは`disabled`に
- adminは「🔓 ロック解除」ボタンで解除可能

### 4-3. 権限による表示制御
- 見えてはいけないデータは**そもそもAPIで返さない**（フロント非表示ではなく）
- 360度の個人スコア（A,B,C列）は被評価者のAPIレスポンスに含めない
- 面談記録は被評価者のAPIレスポンスに含めない

### 4-4. スマホ対応
- サイドバー → ハンバーガーメニュー（☰ボタン）
  - タップでサイドバーがスライドインで表示
  - 背後にオーバーレイ（半透明黒）が表示される
  - オーバーレイタップまたはページ遷移時に自動で閉じる
- タブ → 横スクロール
- テーブル → カード形式に変換
- 行動評価のテーブル → 項目ごとのカード形式

### 4-5. 通知バッジ
- ヘッダーの🔔アイコンに未読通知数を表示
- 通知一覧: 行動評価依頼、目標設定リマインド、評価結果公開

### 4-6. 年度タブ切り替え
- 面談記録画面で年度タブ（2024/2025/2026年度）をクリックすると該当年度のデータに切り替え
- アクティブなタブはオレンジ背景、非アクティブは白背景

### 4-7. 社員詳細の動的データ切り替え
- 社員一覧の行クリックで社員詳細画面に遷移
- プロフィール（アバター・名前・職種・等級など）、評価履歴テーブル、能力推移チャートが選択した社員に合わせて動的に切り替わる
